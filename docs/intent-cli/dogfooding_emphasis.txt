FARLEY/FOWLER END-TO-END TESTING PRINCIPLES
─────────────────────────────────────────────────────

1. TEST THROUGH ACTUAL CLI BOUNDARY
   - Use: gleam run -- <command> <args>
   - NOT: Importing and calling internal functions

2. TEST OBSERVABLE BEHAVIOR
   - What users see: exit codes, JSON output, error messages
   - NOT: How functions work internally

3. TEST FULL WORKFLOWS
   - Complete user scenarios: start → process → finish
   - NOT: Individual function tests

4. DON'T TEST IMPLEMENTATION
   - Black-box testing: CLI as a whole
   - NOT: White-box testing: internals

5. TEST CONTRACTS
   - Exit codes: 0, 1, 3, 4
   - JSON structure: required fields present
   - Error codes: standard messages

6. REAL USER SCENARIOS
   - Happy paths
   - Error recovery
   - Edge cases
   - Real usage patterns
